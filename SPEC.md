# 科技政策综合看板 — 实现规格文档

## TechPolicy Dashboard — Implementation Spec

---

## 1. 项目概述

### 1.1 产品定位

面向**政府科技部门非技术人员**（科委、发改委、工信部等体制内写政策稿子的研究员）的交互式综合看板。用户不是技术背景，需要快速理解前沿技术是什么、和哪些政策相关、在产业里怎么落地、市场资金怎么流动。

副目标用户：金融市场研究员（券商/投行），因为政策效应 → 资本市场反应，看板内容高度重合。

### 1.2 解决什么问题

政策热词 ↔ 具体技术之间缺乏桥梁。写稿子的人看到"具身智能""算力网"这些词，不知道它具体是什么、对应哪些技术、产业落地到哪一步。导致写出来的政策"发空"。

### 1.3 核心设计原则

- **渐进式披露**：关键词 → 点击展开 → 详细内容，不要一次堆文字
- **可视化优先**：图 + 关键词引导，文字藏在下一层
- **四轴关联**：技术 ↔ 政策 ↔ 产业 ↔ 市场，四者互通可跳转
- **面向忙碌的人**：5 分钟内理解一个技术的全貌
- **引导式交互**：不提供自由搜索，而是精心设计热词引导用户探索

## 2. 页面结构

整体是一个**单页应用**，四大模块以 Tab 或侧边栏切换。底部常驻一个**对话框/Agent**。

```
┌─────────────────────────────────────────────────────┐
│  顶部：技术选择（具身智能 / 脑机接口 / 量子计算...）   │
├──────────┬──────────┬──────────┬──────────────────────┤
│          │          │          │                      │
│  模块一   │  模块二   │  模块三   │  模块四              │
│  技术介绍  │  政策动向  │  产业场景  │  市场动向            │
│          │          │          │                      │
├──────────┴──────────┴──────────┴──────────────────────┤
│  底部常驻：对话框 / Agent 引导区                        │
│  [精心设计的热词按钮]  [对话输入框]                      │
└─────────────────────────────────────────────────────┘
```

## 3. 模块一：技术介绍（Tech Explorer）

### 3.1 目标

让非技术人员通过"摸"和"拆"来理解一个技术的结构。不靠文字解释，靠交互探索。

### 3.2 交互层级（三层渐进）

**第一层：主视图**
- 一张技术的可视化大图（SVG）
- 例：具身智能 → 机器人全身图，标注了各个可交互区域
- 图上只显示大类标签（"感知系统""运动控制""决策智能"），不堆细节

**第二层：Hover 拆解**
- 鼠标悬浮到某部件 → 该部件**高亮 + 爆炸拆解**为子组件
- 子组件向外散开，每个子组件旁浮出 2-3 个技术关键词（气泡标签）
- 例：hover"运动控制" → 拆解为：伺服电机、关节控制、路径规划
- 动画用 Framer Motion layout animation，鼠标移走后收回

**第三层：点击下钻**
- 点击关键词气泡 → 展开**技术卡片**（侧边面板或弹窗）
- 卡片内容：
  - 名词解释（100 字以内，通俗语言，非技术人员能看懂）
  - 近期成果（2-3 条，附来源链接）
  - 当前能力边界（能做到什么，做不到什么）
- 卡片底部有"查看相关政策"和"查看产业应用"按钮，跳转到模块二/三

### 3.3 关键交互细节

- Hover 时不要一次性显示太多文字，先显示关键词（3-5 个字），点击后才展开详细内容
- 爆炸动画：部件向外位移 + 缩放，不是真的变形，而是"散开"
- 如果子组件太多，hover 时只显示大类（如"控制"），再 hover 大类才展开具体技术
- 技术卡片支持 ESC 或点击外部关闭

### 3.4 示例数据：具身智能

```
具身智能
├── 感知系统
│   ├── 计算机视觉 — 用摄像头和算法让机器人"看见"
│   ├── 力觉传感器 — 感知接触力的大小和方向
│   └── 触觉反馈 — 模拟人类触觉的电子皮肤
├── 运动控制
│   ├── 伺服电机 — 精确控制转动角度和速度
│   ├── 关节控制 — 多关节协调运动的算法
│   └── 路径规划 — 从 A 到 B 的最优运动轨迹
└── 决策智能
    ├── 大语言模型 — 理解自然语言指令
    ├── 强化学习 — 通过试错学习最优策略
    └── 世界模型 — 对物理世界的内部模拟
```

## 4. 模块二：政策动向（Policy Tracker）

### 4.1 目标

把抽象的政策文件和技术关键词建立可视化的关联。让写稿子的人知道：某个政策热词对应技术的哪个部分。

### 4.2 核心功能

**政策时间轴**
- 横轴为时间（2020-2026），上方浮着政策关键词节点
- 节点按部委颜色编码：科委(蓝) / 工信部(绿) / 发改委(橙) / 国际(灰)
- 点击节点 → 展开政策卡片

**政策卡片**
- 标题 + 日期 + 部委
- 通俗摘要（100 字，非技术人员能看懂）
- 原文链接
- "关联技术"标签（可点击跳转到模块一对应位置）

**政策-技术关联线**
- 从政策卡片拖出一条虚线到模块一中对应的技术节点
- 如果在同一页面不可见，显示"→ 跳转到技术介绍"的链接

**三部门筛选**
- 顶部筛选器：全部 / 科委 / 工信部 / 发改委
- 每个部门有不同的政策特征：
  ```
  科委 → 科研项目、科研设施、技术攻关（散财投资，考核宽松）
  工信部 → 标准化建设、成果转化、产业配套（桥梁角色）
  发改委 → 经济发展、大额投资（最终出钱的人，大金额项目）
  ```

**政策层级展开**
```
十五五规划（最宏观，每个字凝聚大量信息）
  → 部委级规划（科委/工信部/发改委各自的布局）
    → 具体政策文件（芯片法案、算力网规划等）
      → 关联到技术关键词
```

### 4.3 政策数据粒度

不同级别的人需要不同粒度：
- **国家级**：十五五规划、中央经济工作会议（一行字，但凝聚大量信息）
- **部委级**：各部委的具体规划和指导意见
- **地方级**：省市的具体实施方案（如上海市生物医药疗法清单）

黑客松阶段先做国家级 + 部委级。

## 5. 模块三：产业场景（Industry Chain）

### 5.1 目标

展示技术从实验室到产业的完整转化链条。让用户看到一个技术是怎么一步步变成产品的。

### 5.2 核心功能

**产业链视图**
- 一条从左到右的链条：基础研究 → 工程实现 → 零部件 → 整机/系统 → 产业应用
- 每个环节是一个可点击的节点
- 点击节点 → 展示该环节的详情 + 关联技术 + 关联政策

**政策时间轴 + 产业词叠加**
- 底部是政策时间轴（和模块二共用）
- 时间轴上方浮着产业关键词气泡
- 产业词按出现的政策时间点定位在时间轴上
- 例：2026 年 4 月中央经济工作会议提到"算力网" → 时间轴上该位置浮一个"算力网"气泡
- 点击气泡 → 展开该产业词的详情面板

**产业详情面板**
- 文字介绍（什么是算力网）
- 产业链位置（GPU芯片 → 服务器 → 数据中心 → 算力调度 → 应用）
- 关联技术（连线回模块一）
- 关联政策（连线回模块二）

**产业热词来源**
- 数据来源：券商行业报告、研究院报告（这些机构已经做过产业梳理）
- 黑客松阶段手动整理 3-5 个产业热词作为示范

### 5.3 示范数据

```
算力网
  产业链：GPU芯片 → 服务器 → 数据中心 → 算力调度 → AI应用
  关联技术：并行计算、网络通信、分布式存储
  关联政策：东数西算工程、十五五"数字基础设施"

人形机器人
  产业链：减速器 → 伺服电机 → 传感器 → 控制器 → 整机
  关联技术：关节控制、计算机视觉、强化学习
  关联政策：工信部人形机器人指导意见(2023)

生物医药
  产业链：基因编辑 → 药物递送 → 临床试验 → 生产 → 上市
  关联技术：CRISPR、mRNA、AI药物发现
  关联政策：十五五"生物医药"
```

## 6. 模块四：市场动向（Market Trends）

### 6.1 目标

展示技术在资本市场的动态：资金流向哪里、政策出来后市场反应多快、哪个产业环节资本过剩/不足。

### 6.2 核心功能

**政策 → 市场反应时间**
- 政策发布后，市场（尤其是一级市场/创投）多快做出反应
- 可视化：政策节点 → 市场反应节点之间的时间差标注
- 一级市场（创投）反应速度远快于二级市场

**资本流向图**
- 产业链各环节的资本流入/流出可视化
- 可以看出哪个环节资本集中、哪个环节资本不足
- 资本不足的环节 → 政策可以发力的方向（政府补位）

**行业群图（Industry Group Chart）**
- 按行业逻辑组织的技术群/产业群图
- 支持缩放（zoom in/out）
- 时间轴可拖拽，查看不同时间点的状态
- 节点大小代表市场规模，颜色代表资本热度

**数据来源**
- 券商行业报告、研究院报告（这些机构已经做过产业梳理和数据整理）
- 黑客松阶段用静态示范数据

### 6.3 政策与市场的闭环逻辑

```
政策出台 → 资本市场反应 → 资金流向产业链各环节
                                    ↓
                        哪个环节钱多？哪个环节钱少？
                                    ↓
                    钱少的环节 → 政策大手可以发力
                    （直接出政策引导 / 国资暗戳戳补位）
```

## 7. 底部对话框 / Agent

### 7.1 目标

提供一个引导式的交互入口，让用户通过点击热词（而非自由输入）来探索看板内容。

### 7.2 交互设计

**热词引导区**
- 显示 3-5 个精心设计的热词按钮（如"具身智能""算力网""芯片法案"）
- 用户点击热词 → 看板自动跳转到对应的内容位置
- 热词根据当前页面上下文动态变化

**对话框**
- 用户可以输入简短的问题或关键词
- 系统根据输入生成/高亮看板中的相关内容
- 显示思考过程（让用户知道系统在做什么）

**记忆机制**
- 记住用户在当前会话中探索过的节点
- 用户可以回退到之前的某个探索节点
- 支持"开分叉"：从某个历史节点开始新的探索路径
- 记忆不需要很复杂，因为用户交互就几句话

**引导流程示例**
```
1. 用户点击热词"具身智能"
2. 看板跳转到技术介绍模块，展示机器人全身图
3. 对话框提示："这是具身智能的技术结构。hover 各部件探索。"
4. 用户 hover 运动控制 → 看到关键词
5. 用户点击"伺服电机" → 技术卡片展开
6. 对话框提示："想看相关政策吗？" [查看政策] [查看产业]
7. 用户点击 [查看政策] → 跳转到模块二，高亮相关节点
```

## 8. 数据模型

### 8.1 核心实体

```typescript
// 技术
interface Technology {
  id: string;
  name: string;                    // "伺服电机"
  description: string;             // 通俗解释，100字以内
  category: string;                // "运动控制"
  parentId: string | null;         // 所属大技术
  capabilitySummary: string;       // 当前能力边界
  recentAchievements: {            // 近期成果
    title: string;
    date: string;
    source: string;                // 来源链接
  }[];
  position: { x: number; y: number; };  // 在可视化图中的坐标
  relatedPolicies: string[];       // 关联政策ID
  relatedIndustries: string[];     // 关联产业ID
}

// 政策
interface Policy {
  id: string;
  title: string;
  country: string;                 // "中国" / "美国"
  department: string;              // "科委" / "工信部" / "发改委"
  level: string;                   // "national" / "ministerial" / "local"
  date: string;
  summary: string;                 // 通俗摘要，100字
  fullTextUrl: string;
  relatedTechnologies: string[];   // 关联技术ID
  relatedIndustries: string[];     // 关联产业ID
  marketReactionDays?: number;     // 政策出台后市场反应天数
}

// 产业
interface Industry {
  id: string;
  name: string;                    // "算力网"
  description: string;
  nationalPlanRef: string;         // "十五五"
  chainStages: {                   // 产业链各环节
    name: string;                  // "GPU芯片"
    technologyIds: string[];       // 关联技术ID
  }[];
  relatedPolicies: string[];
  capitalFlow?: {                  // 资本流向数据
    stage: string;                 // 产业链环节
    amount: number;                // 资本流入金额
    trend: "up" | "down" | "stable";
  }[];
}

// 会话记忆
interface SessionMemory {
  visitedNodes: {                  // 用户探索过的节点
    nodeId: string;
    nodeType: "tech" | "policy" | "industry";
    timestamp: number;
  }[];
  branches: {                      // 分叉点
    fromNodeId: string;
    label: string;
  }[];
}
```

### 8.2 关联关系

四个模块通过 `relatedTechnologies`、`relatedPolicies`、`relatedIndustries` 三个字段互相连接，形成知识图谱。任意模块的节点都可以跳转到其他模块的关联节点。

## 9. 技术架构

### 9.1 技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| **前端框架** | Next.js 15 (App Router) | 可以做纯前端，部署简单 |

### 9.3 关键组件实现思路

#### TechDiagram — SVG 热区交互

```
底层：SVG 图片（机器人全身图）
中层：透明 SVG <path> 热区（每个部件一个 path）
顶层：Framer Motion 动画层

hover 热区 → 该 path 高亮 + 触发 ExplodeAnimation
点击热区 → 展开 KeywordBubble
点击气泡 → 展开 TechCard（侧边面板滑入）
```

#### ExplodeAnimation — 爆炸拆解

```
使用 Framer Motion 的 layout + animate:
1. 原始状态：子组件在父部件内紧凑排列
2. hover 触发：子组件 animate 到新的 position（向外散开）
3. 每个子组件旁 AnimatePresence 挂载 KeywordBubble
4. 鼠标移走：子组件 animate 回原位，KeywordBubble exit
```

#### PolicyTimeline — 政策时间轴

```
底部：D3 绘制的时间轴（横轴 2020-2026）
上方：政策节点（按日期定位，按部委着色）
节点可点击 → 展开 PolicyCard
从 PolicyCard 可拖出关联线到技术节点
```

#### ConnectionLines — 关联线

```
SVG overlay 覆盖整个看板
从源节点 (x1,y1) 到目标节点 (x2,y2) 画虚线
stroke-dasharray: "5,5"
鼠标 hover 关联线 → 显示关联说明 tooltip
```

#### CapitalFlow — 资本流向

```
产业链各环节作为节点排列
节点之间的连线粗细代表资本流量
节点颜色：红=资本过剩，绿=适中，黄=不足
可选：桑基图（Sankey）展示资金来源→去向
```

## 10. 视觉风格

- **深色主题**（深灰/深蓝背景），适合长时间观看
- **霓虹风格的数据可视化**（参考 Bloomberg Terminal 风格）
- 关联线用半透明虚线，不抢视觉焦点
- 关键词气泡用圆角卡片 + 微妙的发光效果
- 动画克制，不要过度炫技，服务于信息传达

## 11. 示范数据清单

黑客松阶段需要准备以下静态数据：

### 技术：具身智能
- 3 个子系统（感知/控制/决策），每个 2-3 个子技术
- 每个子技术的通俗解释 + 近期成果 + 能力边界

### 政策：8-10 条
- 中国：十五五规划、工信部人形机器人指导意见、东数西算、中央经济工作会议
- 美国：芯片法案、AI行政令
- 每条附通俗摘要 + 关联技术

### 产业：3-4 个
- 算力网、人形机器人、生物医药、商业航天
- 每个含产业链环节 + 关联技术 + 关联政策

### 市场数据
- 2-3 个产业链的资本流向示意数据
- 政策→市场反应时间示意数据

### Agent 热词
- 5-8 个精选热词按钮，覆盖不同模块的入口
